# CLAUDE.md — BERMONT 브랜드 홈페이지

이 문서는 BERMONT 웹사이트 개발의 단일 기준(source of truth)이다.
모든 구현 결정은 이 문서를 따른다. 충돌하는 요청이 오면 구현 전 확인 질문을 할 것.
승인된 디자인 프로토타입: `docs/bermont-prototype.html` (단일 파일 HTML 산출물).
비주얼/카피/톤은 이 프로토타입이 기준이며, 프레임워크로 이식할 때 임의 변경하지 않는다.

---

## 1. 브랜드 개요

- **브랜드명**: BERMONT
- **사로건**: `FABRIC FIRST` — 영문 고정, 번역하지 않는다. 타이포로써는
  FABRIC(원단) / FIRST(온라인 스트리트) 2단 표현 처리.
- **구 태그라인 "적이 지나갈 때, 아름이 남는다"는 폐기됨. 어디에도 사용 금지.**
- **포지셔닝**: 원단 스펙(무게, 혼용률, 수축률, 가공)을 정면 공개하고
  숫자로 증명하는 프리미엄 어패럴. 오버사이즈는 핏 시스템.
- **주 타겟**: 30–40대 포함 성인 전반. 단, 사이트의 디자인 언어는
  힙한 어패럴/스트리트웨어 트렌드를 적극적으로 따른다 (다크 문법, 헤비 타입, 터커 등).
- **톤**: 형용사 대신 데이터. "부드러운", "고급스러운" 류의 형용어 금지.
  "320 GSM", "수축률 <1.5%" 처럼 측정값으로 말한다.

## 2. 브랜드 아키텍처

**구조: 브랜드드 하우스 (Branded House).**

- 현재 버티컬: **어패럴 단독**. 사이트에 존재하는 유일한 사업.
- 미래 버티컬(F&B / Spirits / Financial)은 **완전 숨김 처리**:
  - 페이지, 메뉴, 푸터 링크, 텍스트 문구 일절 노출 금지 (이전 버전의 "What's Next" 티저도 폐기)
  - 코드 레벨 준비만 허용: 라우트 네임스페이스, 색상 시스템 내 토큰 몇몇
- 금융 버티컬의 향후에도 이 텍스트 넣지 않는다 (별도 법인/도메인/코드베이스 전제).
- 주류는 국내 통신 판매 불가 → 향후 부이든 스토리텔링 전용.

## 3. 정보 구조 (IA) & 라우팅

```
/                  — 홈 (히어로 → 터커 → 드롭 → Fabric Lab → 핏 시스템 → 뉴스레터)
/apparel           — 샵 (전체 제품)
/apparel/[slug]    — 제품 상세
/fabric-lab        — 원단 스펙시트 아카이브 (브랜드 신뢰 자산)
/fit               — 핏 시스템 가이드 (Relaxed / Over / Heavy)
/journal           — 에디토리얼
/journal/[slug]    — 아티클
```

- 헤더 네비: Shop / Fabric Lab / Fit / Journal + 언어 스위처 + 카트.
- 미래 버티컬용 라우트는 만들지 않는다. 네임스페이스 패턴만 유지.

## 4. 다국어 (i18n) — 필수 요구사항

**지원 언어 5종: 한국어(ko, 기본) / 영어(en) / 러시아어(ru) / 스페인어(es) / 일본어(ja)**

- 라이브러리: `next-intl` (App Router 기준) 또는 동급. 하드코딩 문자열 금지 —
  모든 UI 텍스트는 로케일 사전을 통해서만.
- 프로토타입의 `I18N` 객체가 초기 번역 사전이다. 키 구조를 그대로 이어받을 것.
- 라우팅: `/{locale}/...` prefix 방식 (`/ko`, `/en`, `/ru`, `/es`, `/ja`).
  기본 로케일(ko)은 prefix 생략 가능하게 설정.
- 언어 선택은 쿠키/스토리지에 저장, 재방문 시 유지.
- `html lang` 속성 로케일 동기화, hreflang 태그 필수 (SEO).
- 번역하지 않는 것: 사로건 `FABRIC FIRST`, 제품명, 소재 단위(GSM, g/m² 등),
  드롭 코드 (DROP 001 — SS26).
- CJK(ko/ja)와 키릴(ru) 렌더링 확인 — Archivo는 라틴 전용이므로
  ko/ja/ru 디스플레이 타이포는 시스템 폴백 또는 Noto Sans 계열로 폴백 체인 구성.

## 5. 디자인 시스템 & 토큰

**원칙: 색/타이포/간격은 토큰으로만. 하드코딩 hex 금지. 토큰명은 시맨틱으로.**

### 컬러 (프로토타입 확정 값)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` | `#0B0B0C` | 기본 배경 |
| `--surface` | `#141416` | 카드/패널/티커 |
| `--line` | `#232326` | 보더, 디바이더 |
| `--accent` | `#C6F33A` | 네온 라임 — 포인트, 글로우, CTA |
| `--accent-dim` | `rgba(198,243,58,.14)` | 은은한 발광 배경 |
| `--text` | `#EDEDE8` | 본문 |
| `--muted` | `#8A8A85` | 보조 텍스트 |

- 네온은 액센트 전용. 면적 사용 금지. 글로우는 `drop-shadow`/`box-shadow`로 절제해서.

### 타이포그래피

| 역할 | 폰트 | 설정 |
|---|---|---|
| Display | **Archivo** (variable) | wght 800–900, wdth 62–112. 헤드라인은 uppercase + wdth 112, 배경 숫자 그래픽은 wdth 62 (컨덴스드) |
| Body | Inter | 400–600 |
| Technical | IBM Plex Mono | 스펙 수치, 칩, 라벨로우, 티커, 버튼 라벨 |

- **Fraunces(세리프)는 폐기됨. 사용 금지.**
- 원단 데이터는 무조건 모노스페이스 — 브랜드 시그니처.
- 유동 타이포: `clamp()` 기반. 히어로 h1은 `clamp(52px, 9.5vw, 138px)`.

### 시그니처 요소 (유지 필수)

1. **배경 아웃라인 "88"**: 히어로의 거대 스트로크 숫자, 하단부만 accent 스트로크 + 글로우.
2. **행택(Hang Tag) 스펙시트**: 원단 항공 태 + 네온 스탬핑 포인트 + 점선 구분 태.
   Fabric Lab과 제품 상세의 공통 컴포넌트로 사용.
3. **무한 스크롤 티커**: 스펙 키워드 순환. `prefers-reduced-motion` 시 정지.

### 모션/접근성

- 스크롤 리빌: IntersectionObserver, `opacity + translateY`, 1회성.
- `prefers-reduced-motion: reduce` 시 티커/리빌/smooth-scroll 전부 비활성화 (프로토타입 구현 그대로).
- 키보드 포커스: `:focus-visible`에 accent 아웃라인.
- 다크 배경 대비: muted 텍스트는 WCAG AA 이상 유지 확인.

## 6. 기술 스택

- **프레임 워크**: Next.js (App Router) — 멀티디바이스(반응형 웹) 기준.
  네이티브 앱은 버전 이후, 반응형으로 커버.
- **반응형 브레이크포인트**: 모바일 1열(≤720px) / 태블릿 2열(≤960px) / 데스크톱 3열.
- **스타일**: CSS 변수 토큰 + Tailwind 사용 시 토큰을 theme으로 매핑.
- **i18n**: next-intl (4번 참조).
- **콘텐츠**: 초기 로컬 MDX/JSON → 확장 시 헤드리스 CMS.
- **커머스**: 외부 연동 (Shopify 헤드리스 또는 카페24). 결제/재고 직접 구현 금지.
- **폰트 로딩**: `next/font` 사용, Archivo/Inter/IBM Plex Mono + CJK/키릴 폴백.

## 7. 이번 작업 순서 (권장)

1. Next.js + next-intl 세팅, 5개 로케일 라우팅 골격
2. 디자인 토큰을 글로벌 CSS 변수/theme으로 정의
3. 프로토타입을 컴포넌트로 분해:
   `Header(LangSwitcher 포함) / Hero / Ticker / ProductCard / DropGrid / HangTagSpec / FitCard / Newsletter / Footer`
4. `I18N` 사전을 로케일별 메시지 파일로 분리 (`messages/ko.json` 등)
5. 상품 데이터를 정적 JSON으로 분리 → `/apparel` 목록/상세 연결
6. 접근성/반응형/reduced-motion 회귀 확인
